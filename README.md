# Apk Re-Signer
<img height="300" alt="apkresigner@2x" src="https://github.com/user-attachments/assets/ad68b528-1ea9-416b-8209-9ca120c438f2" />

A desktop tool and a workaround for re-sign the .apk files generated by Visual Studio 2026 v18.6.3-v18.7.1 that might lead to -124 failure in installation on android system. 

> [!TIP]
>
> The -124 failure is described as below:
>
> `-124: Failed parse during installPackageLI: Targeting R+ (version 30 and above) requires the resources.arsc of installed APKs to be stored uncompressed and aligned on a 4-byte boundary`
>
> More details can be checked at: https://github.com/Blatke/Workaround-for-Visual-Studio-Generated-MAUI-APK-Installation-Failure-Error--124-

## Download
https://github.com/Blatke/Apk-Re-Signer/releases
## Requirements for Installation
1. Windows 7.0+.
2. .NET 10.0+
## Requirements for Processing
1. Java.
2. Android Studio.
3. 7-Zip.
4. Your .apk file generated by Visual Studio.
5. Your keystore set by Visual Studio.
